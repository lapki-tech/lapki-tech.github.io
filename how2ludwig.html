How to get started with Machine Learning using Uber Ludwig

Imagine if Machine Learning required no PhD in Computer Science. Imagine if somebody without any special AI knowledge could put an AI into any kind of application without any hassle. There is a solution for that. I have used Uber Ludwig both in research and industrial tasks and it is very well suited for lightning speed prototyping and testing simple hypotheses. Ludwig is a perfect first step in solving your Machine Learning task, before you try implementing something complicated in Pytorch or Tensorflow. Chances are, you won't need anything else. Basically, Uber Ludwig is CreateML before CreateML was mainstream (it still isn't mainstream though). It's like CreateML but based.

It has no fancy graphical interface and no restrictions like "you can use only fancy apple hardware". It is marketed as "no-code ML" but IN IMPERIAL RUSSIA "no-code" means "not cool enough". We'll do that the proper way, with a lot of python.

What I'll do is simple: I'll make a solution for a Kaggle problem with ludwig to show the basic steps. You can use this example as a guideline to use Ludwig and there is a good probability you wouldn't need any other material but this notebook and Ludwig official documentation.

Now let's go to kaggle and get our dataset downloaded. Our task is Mechanisms of Action prediction. Key idea in this competition (which ended six-ish month ago) is to guesstimate the way how a certain drug works from the results of some biological experiments. That explanation is good enough for today, you can check out the paper for the dataset by following the link in the description.

We have the following files in the dataset, we won't use train_drug.csv and train_targets_nonscored, so we're left with:

train_features.csv - Features for the training set. Features g- signify gene expression data, and c- signify cell viability data. cp_type indicates samples treated with a compound (cp_vehicle) or with a control perturbation (ctrl_vehicle); control perturbations have no MoAs; cp_time and cp_dose indicate treatment duration (24, 48, 72 hours) and dose (high or low).

train_targets_scored.csv - The binary MoA targets that are scored.

test_features.csv - Features for the test data. You must predict the probability of each scored MoA for each row in the test data.

sample_submission.csv - A submission file in the correct format.

The columns have different types. Ludwig offers a type abstraction to describe the dataset for the model. There are a lot of types (numeric, image, category, binary, text, sound and so on). The types determine what kind of a model would be used, Ludwig offers some state-of-the-art models for image data.

We've covered everything that would help you try machine learning fast with ludwig. Except for one tiny thing... How to use the model? Imagine that you would like to load the model inside an app. What should you do?

Ludwig automates saving the snapshots of the model and keeps (by default) the best model discarding all the rest. It saves the model in the results folder, making a folder for each experiment.

The experiment folder has three items in it:

  1. description.json - the full description of the training. You can get the config out of the config field, and see some metadata (version, random seed etc) in the rest of the fields.
  2. training_statistics.json - the history of training for each of the targets, the one we've visualized above.
  3. model folder - the weights and other data required to use the model.

We load the model by providing the path to model folder for the load method of the LudwigModel class.
